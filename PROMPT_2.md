# PROMPT.md — Refactor Existing Customer Support Website Into a Call-Centered Support Website

## IMPORTANT

You are working on an EXISTING project that is already in a good state.

Your job is NOT to rebuild everything from scratch.
Your job is to REFINE, REPOSITION, and UPGRADE the existing project.

The current website feels too focused on:
- text support
- messages
- chat-based communication
- written replies

This must be changed.

The website must now be primarily positioned around:
- phone-based support
- call handling
- inbound and outbound calls
- callback requests
- customer service by phone
- call center / cloud telephony style support
- voice-first customer communication

Forms can still exist and should remain useful, but the MAIN essence of the website must clearly communicate that the company’s core offering is CALL SUPPORT, not text support.

A form can still be used for:
- consultation requests
- service inquiries
- callback requests
- lead capture
- demo booking

But the WEBSITE’S CORE POSITIONING must be about CALLS.

---

## MAIN GOAL

Take the existing support SaaS / customer support website and transform it so that:

1. The messaging/chat-centered positioning becomes call-centered.
2. The text, headings, section purposes, and value proposition reflect phone support instead of message support.
3. Illustrations and product visuals are updated so they look relevant to calls, call center work, phone agents, call routing, voice support, call analytics, etc.
4. Add more “live” and real-feeling images where appropriate.
5. Perform a visual QA / polish pass so everything looks aligned, balanced, clean, and professionally finished.

Do not break the existing project structure unless necessary.

---

## CORE POSITIONING CHANGE

The new product identity should feel like a modern platform for:

- call support teams
- customer service by phone
- business phone support
- cloud calling
- call routing
- smart call handling
- callback workflows
- support agents handling voice conversations
- AI-enhanced call support
- call analytics and insights

The website should clearly communicate that the main service is:

“Professional customer support centered around calls.”

Possible positioning ideas:
- Voice-first customer support
- Business call support made simple
- Smarter customer service through calls
- Human-first phone support for modern teams
- Call support software for teams that want clarity and speed

Do NOT let the website still feel like it is mostly about chat or shared inbox messaging.
If chat appears anywhere, it must be secondary, optional, or supportive — not the main story.

You must check and improve:

### Layout consistency
- spacing between sections
- consistent paddings and margins
- balanced vertical rhythm
- consistent container widths
- proper alignment of text and images
- consistent card heights where appropriate

### Typography consistency
- heading sizes
- paragraph widths
- line heights
- visual hierarchy
- button labels and sizing
- text wrapping problems
- awkward line breaks

### Component polish
- buttons alignment
- form alignment
- icon placement
- card padding
- grid consistency
- section headers alignment
- CTA blocks alignment
- footer spacing

### Responsive quality
Check mobile, tablet, and desktop.
Fix:
- broken layouts
- overlapping blocks
- awkward image cropping
- unequal card heights causing messy appearance
- buttons wrapping poorly
- too much or too little spacing on mobile
- horizontal overflow

### “Make everything look straight”
This includes:
- visually balanced composition
- no crooked spacing
- no random misalignment
- no inconsistent radius/shadow usage
- no broken grid feeling
- no section that looks off compared to the rest of the site

The result should feel polished and production-ready.
